## Overview

This repository contains example Linux kernel modules based on the
*Linux Device Drivers, 3rd Edition (LDD3)* educational codebase.

The repository is primarily used to build and experiment with out-of-tree
kernel modules in embedded Linux systems.

The repository is used as a source of example kernel modules integrated into 
my Yocto and Buildroot embedded Linux projects.

It is used as a learning reference for Linux kernel module development,
including character drivers, debugging modules, and example kernel subsystems.

---

## Kernel Modules

This repository contains multiple example kernel modules used for educational purposes.

### Actively Used Modules

Scull and misc-modules drivers are used in the final embedded Linux images generated by the Yocto and Buildroot projects.

#### scull (Simple Character Utility for Loading Localities)

A RAM-based character device driver that simulates a simple disk-like device.

Features:
* Implements a character device driver
* Provides multiple device instances (`/dev/scull0` to `/dev/scull3`)
* Stores data in kernel memory
* Demonstrates file operations (`read`, `write`, `open`, `release`)
* Demonstrates device registration and minor number handling

---

#### misc-modules

A collection of simple example kernel modules used for learning and debugging.

Utilized modules include:

* `hello` – basic kernel module that prints messages using printk
* `faulty` – intentionally causes a kernel crash (used for oops analysis)

These modules are used to demonstrate:
* Kernel module loading/unloading
* Kernel logging
* Kernel crash and debugging analysis

## Additional Example Modules

The repository also contains additional LDD3 example kernel modules such as:

* `seq` - Demonstrates the Linux seq_file interface for sequential virtual file output.
* `jiq` - Shows different mechanisms for deferred work and scheduling in the kernel.
* `sleepy` - Demonstrates blocking I/O and wait queue usage.
* `complete` - Demonstrates synchronization using kernel completions.
* `silly` - Example module illustrating concurrent access and synchronization concepts.
* `kdatasize` - Displays sizes of fundamental kernel data types on the target architecture.
* `kdataalign` - Demonstrates memory alignment and padding of kernel data structures.
* `jit` - Demonstrates timing behavior using jiffies and kernel timers.

These modules are included for educational reference but are not used in the final system images generated by the Yocto and Buildroot projects.

## Building

All modules are implemented as out-of-tree Linux kernel modules and are built using the Linux kernel build system.

The modules may be compiled either directly against a Linux kernel source tree or integrated into an embedded Linux build system such as Yocto or Buildroot.

For example, a module may be compiled directly against the currently running kernel using:

```sh
make -C /lib/modules/$(uname -r)/build M=$(pwd) modules
```

Refer to my Yocto and Buildroot project repositories for complete build and integration details.

## Usage

Only the `scull` and `misc-modules` components from this repository were integrated into the final embedded Linux systems developed using Yocto and Buildroot.

### Yocto Integration

The modules were packaged through two custom BitBake recipes and included in a custom ARM64 Linux image.

During system startup:

* The SCULL driver is loaded automatically.
* The hello and faulty modules are loaded automatically.
* Device nodes required by the drivers are created through SysV init scripts.

### Buildroot Integration

The modules were packaged through one custom Buildroot package and included in a custom ARM64 Linux image.

During system startup:

* The SCULL driver is loaded automatically.
* The hello and faulty modules are loaded automatically.
* Device nodes are created and cleaned up through SysV initialization scripts and root filesystem overlays.

The remaining modules contained in this repository were retained as educational reference material and were not included in the final embedded Linux images in Yocto and Buildroot projects.
